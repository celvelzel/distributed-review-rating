# Feasibility & Performance Assessment: Graph Feature Expansion

## Current Baseline
- Best Kaggle: 0.61734 (DeBERTa VE 90% + Ridge 10%)
- Stats Ridge OOF: 0.8076 (11 features, pandas-based)
- GCN embeddings: OOF 1.4186 (weak, not useful)

---

## Feature Group Assessment

### 1. Sentiment Features (17 dimensions)

**Source**: `code/features/sentiment.py` — already implemented, reads from parquet

| Aspect | Assessment |
|--------|-----------|
| **Implementation** | Code exists but requires `artifacts/etl/train.parquet` (not available locally). Need CSV adapter. |
| **Dependencies** | `nltk` (VADER), `textblob` — may not be installed |
| **Runtime** | ~30-60s for 3M rows (multiprocessing, 8 workers) |
| **Memory** | ~500MB (text processing) |
| **Expected OOF impact** | Medium. Sentiment correlates with rating but DeBERTa already captures this semantically. Ridge on sentiment alone: ~1.0-1.2 OOF. |
| **Risk** | Low — features are safe (no target dependency) |
| **Feasibility** | ✅ High — adapt to CSV input, ~30 min implementation |

### 2. Rating Deviation Features (6 dimensions)

**Source**: `code/features/rating_deviation.py` — K-Fold OOF, leakage-free

| Aspect | Assessment |
|--------|-----------|
| **Implementation** | Code exists but requires `artifacts/etl/train.parquet`. Need CSV adapter. |
| **Dependencies** | Only `sklearn` (KFold) — already available |
| **Runtime** | ~10-20s for 3M rows (5-fold split + groupby) |
| **Memory** | ~200MB |
| **Expected OOF impact** | High. User leniency/harshness is a strong predictor. These features capture "how much does this user deviate from the norm" — exactly what the stats features miss. |
| **Risk** | Low — K-Fold OOF prevents leakage |
| **Feasibility** | ✅ High — adapt to CSV, ~20 min implementation |

### 3. Store Metadata Features (4 dimensions)

**Source**: `code/features/product_metadata.py` — already implemented

| Aspect | Assessment |
|--------|-----------|
| **Implementation** | `extract_store_features()` reads from prodInfo. Just need to call it on CSV. |
| **Dependencies** | None extra |
| **Runtime** | <1s (simple groupby on 213K products) |
| **Memory** | Negligible |
| **Expected OOF impact** | Low-Medium. Store reputation captures product quality proxy. |
| **Risk** | Very low |
| **Feasibility** | ✅ Very high — 5 min implementation |

| Feature Set | Est. OOF RMSE | vs Current (0.8076) | Notes |
|-------------|--------------|---------------------|-------|
| Stats only (current) | 0.8076 | baseline | 11 features |
| + Sentiment | ~0.75-0.78 | -3~7% | DeBERTa already captures sentiment |
| + Rating deviation | ~0.76-0.79 | -2~6% | Strong but correlated with stats |
| + Store metadata | ~0.79-0.80 | -1~2% | Weak signal |
| + User-category deviation | ~0.77-0.79 | -2~5% | New interaction signal |
| **All combined** | **~0.72-0.76** | **-6~11%** | Diminishing returns likely |

**Kaggle projection** (assuming OOF→Kaggle gap of ~0.45):
- Current stats Ridge: OOF 0.8076 → Kaggle ~0.36 (if it worked standalone)
- Expanded stats: OOF 0.74 → Kaggle ~0.29 (if it worked standalone)
- **But**: Ridge on stats alone won't beat DeBERTa. The value is in **blending** with DeBERTa.

**Realistic blended projection**:
- DeBERTa VE (0.617) + expanded stats Ridge 10% → **0.610-0.615** (1-2% improvement)

## Risk Assessment

| Risk | Likelihood | Impact | Mitigation |
|------|-----------|--------|------------|
| Sentiment slow on 3M rows | Medium | Low | Parallel processing, already implemented |
| Memory OOM with expanded features | Low | Medium | Use float32, process in chunks |
| New features don't improve OOF | Medium | Low | Still useful for ensemble diversity |
| Feature leakage | Low | High | K-Fold OOF for deviation features |

## Recommendation

**Proceed with Phase 1 (quick wins) only.** The 30-minute investment gives us store metadata (4d) + rating deviation (6d) + user-category deviation (~3d) = 13 new features with minimal risk. If these improve OOF, then invest in Phase 2 (sentiment). Don't invest in Phase 2 until Phase 1 is validated.
