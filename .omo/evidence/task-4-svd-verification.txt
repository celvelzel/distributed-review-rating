## Notes on PySpark vs sklearn
- PySpark 3.4.1 does NOT have TruncatedSVD in pyspark.ml.feature
- PySpark RowMatrix.computeSVD fails on 50K-feature matrices:
  - Gram matrix (50K × 50K) is ~20GB, exceeds local mode capacity
  - ARPACK convergence issues with Python Vector serialization overhead
- sklearn.TruncatedSVD uses scipy.sparse ARPACK solver natively on sparse matrices
- This is the standard approach for TF-IDF SVD in production NLP pipelines
- PySpark was still used for ETL data loading (train.parquet)
