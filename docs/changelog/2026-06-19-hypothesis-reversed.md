# 2026-06-19: 假设被推翻 — 3M 数据并非有害

## 背景

基于之前的分析，我们假设 "3M 数据导致过拟合，1M 数据更好"。为了验证这个假设，我们进行了公平对比实验。

## 实验设计

| 实验 | 数据 | 脚本 | 配置 |
|------|------|------|------|
| 1M Old | 1M | 旧脚本 | 5f×5e |
| 1M Fair | 1M | `deberta_base_full.py` | 3f×3e |
| 3M | 3M | `deberta_base_full.py` | 3f×3e |

## 实验结果

| 模型 | 数据 | 配置 | OOF RMSE | Kaggle |
|------|------|------|----------|--------|
| 1M Old | 1M | 5f×5e | 1.117 | **0.617** |
| **1M Fair** | **1M** | **3f×3e** | **1.298** | **1.536** |
| 3M | 3M | 3f×3e | 1.137 | 0.681 |

## 关键发现

1. **假设被推翻**: 3M 数据并非有害！
   - 1M + 3f×3e = Kaggle 1.536 (极差)
   - 3M + 3f×3e = Kaggle 0.681
   - 3M 数据实际上有帮助！

2. **1M Old 的成功来自训练配置**:
   - 1M Old 使用 5f×5e 配置
   - 1M Fair 使用 3f×3e 配置
   - 差距: Kaggle 0.617 vs 1.536

3. **5f×5e 配置是关键**:
   - 更多折数 (5 vs 3)
   - 更多 epoch (5 vs 3)
   - 可能有其他差异 (学习率调度、正则化等)

## 下一步

1. **用旧脚本 (5f×5e) 训练 3M 数据**
   - 如果 Kaggle < 0.617 → 证明 3M 数据 + 5f×5e 配置更优
   - 如果 Kaggle ≈ 0.617 → 证明数据量无影响
   - 如果 Kaggle > 0.617 → 证明 1M 数据更好

2. **调查为什么 5f×5e 配置更优**
   - 更多折数 → 更好的泛化
   - 更多 epoch → 更充分的训练
   - 学习率调度 → 可能更优

3. **更新策略**
   - 放弃 "3M 数据有害" 的假设
   - 专注于训练配置优化
   - 考虑用 5f×5e 配置训练更大的模型
