COMP5434 Project  1. Overview This project requires teams to develop a solution for predicting review ratings using a provided dataset. The objective is to build a method that is both accurate and computationally efficient, leveraging big data parallel/distributed processing systems.  2. Deadlines a. Kaggle Predictions Submission: 11:59 PM, July 1st, 2026 b. Presentation slides, report and code submission on blackboard: 11:59 PM, July 1st, 2026  3. Team Requirements Students should participate in this project in teams. Each team should have a voluntary coordinator for administrative purposes. The coordinator should fill in a form (PolyU Connect accounts required, https://forms.office.com/r/3GqN7HjDri) to register the team on or before 11:59 PM 12th June, 2026. Students not in a team by this deadline will be assigned randomly.  4. Data Description  For the data, we provide its training data (train.csv), extra product information (prodInfo.csv), testing data (test.csv), and submission sample (sample_submission.csv).  You need to download the dataset and upload your prediction results at the specific Kaggle competition:  https://www.kaggle.com/t/9e897d08dba249bb8a1312666e8ef8fd   Your goal is to train a model to predict the rating of each review.  Training data: train.csv - Each row corresponds to a single review and contains the following fields: id (the id of the review), title (the review title), comment (the review text), user_id (ID of the user who posted the review), prod_id (ID of the reviewed product), parent_prod_id (Parent ID of the product. For example, same products with different colors share identical parent ID), time (timestamp when the review was posted), votes (number of users who found the review helpful), purchased (whether the user purchased the product), and ground-truth rating (an integer from 1 to 5). - The last cell of each row (“rating” column) is the prediction target  - Training data includes training records with ground truth. Extra product information prodInfo.csv - Each row corresponds to a product and contains the following fields: id (row number of this product in this file), parent_prod_id, main_category (the category of the product), price, title (name of the product), features (description of the product), store (the store selling the product), rating_number (number of ratings received for this product). You can use train.csv and prodInfo.csv together to train your solution.  Redistributing this file (including partially) to CourseHero or other public websites is strictly prohibited.
