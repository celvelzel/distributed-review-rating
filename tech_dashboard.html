✨ 核心发现
当前最佳 Kaggle 分数: 0.79012 TF-IDF 5K + 正则化 LightGBM (127 leaves, subsample=0.8, colsample=0.8)。相比 Stage 0 基线 (0.80107) 改进 1.4%。
关键发现: 目标泄漏是主要问题 统计特征 (user_te, prod_te, avg_rating) 存在目标泄漏, 导致本地 CV 好 (0.55) 但 Kaggle 差 (1.2-1.6)。纯 TF-IDF 特征泛化最好。
TF-IDF 配置仍有优化空间 当前仅使用词级 (1,1) n-gram。添加字符级 n-gram (char_wb, 2-5) 可捕捉拼写变体和词缀模式, 预计提升 5-10%。
MLP 性能不佳的原因 DeBERTa 冻结嵌入 + LightGCN 的 MLP OOF RMSE = 1.152。原因: 嵌入未微调, 且与泄漏特征混合使用。需要单独使用 BERT 嵌入。
改进路线: 三阶段优化 阶段1: 优化 TF-IDF 配置 (字符级 n-gram + 文本预处理)。阶段2: 多模型集成 (XGBoost + Ridge)。阶段3: SentenceTransformer 嵌入。

🧪 实验总结

已尝试的方法:

  • TF-IDF 特征 (5K, 8K, 10K, 15K, 20K)
  • N-gram 范围: (1,1), (1,2), (1,3)
  • LightGBM 超参数调优 (leaves, lr, subsample, colsample)
  • 添加泄漏-free 特征 (temporal, text_length, votes)
  • 集成多个模型 (equal weight, weighted, median)
  • 混合最佳模型与 Stage 0 (不同比例)
  • 后处理 (四舍五入, 不同 clipping, 添加噪声)
  • 正则化 (subsample=0.7-0.8, colsample=0.7-0.8, reg_alpha/lambda)

关键结论:

  • TF-IDF 特征是最重要的
  • 添加额外特征反而降低性能
  • 正则化 (subsample, colsample) 有助于泛化
  • 集成/混合没有显著改善

根因分析:

  • 统计特征 (user_te, prod_te) 存在目标泄漏 → Kaggle 分数 1.2-1.6
  • DeBERTa 嵌入 + MLP 效果差 (OOF RMSE 1.152) → 嵌入质量不足
  • TF-IDF 维度不足 (5K) → 无法捕捉所有文本模式
  • 缺乏字符级特征 → 无法处理拼写变体和词缀
🎯 下一步计划

为了缩小与竞争对手的差距, 需要尝试:

  1. 字符级 TF-IDF: 使用 char_wb 分析器, n-gram (2,5), 捕捉拼写变体
  2. 更好的文本预处理: 小写化、移除特殊字符、词干提取
  3. 增加 TF-IDF 维度: 从 5K 增加到 20K-50K
  4. XGBoost: 尝试不同的梯度提升算法
  5. SentenceTransformer: 使用 all-MiniLM-L6-v2 提取语义嵌入
  6. 多模型集成: TF-IDF + LightGBM + XGBoost + Ridge
  7. DeBERTa 微调: 直接微调预训练语言模型

预期改进:

  • 阶段 1 (优化 TF-IDF): 0.79 → 0.77-0.78
  • 阶段 2 (多模型集成): 0.77 → 0.75-0.76
  • 阶段 3 (BERT 嵌入): 0.75 → 0.70-0.72
  • 阶段 4 (Transformer 微调): 0.70 → 0.65-0.68
