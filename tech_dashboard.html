✨ 核心发现
🏆 方差扩展是关键突破 (最重要的单一技术) DeBERTa 预测严重压缩: 预测 std=0.825 vs 实际 target std=1.422 (仅捕获 58% 方差)。通过线性缩放匹配目标分布，Kaggle 从 0.638 → 0.617 (单技术提升 3.3%)。
scale = target_std / pred_std = 1.422 / 0.825 = 1.72
pred_calibrated = (pred - pred_mean) × scale + target_mean
为什么有效: Kaggle 测试集分布与训练集相似。DeBERTa 捕获了排序信号但压缩了幅度。方差扩展恢复了正确的尺度。
关键证据: VE 90% + Ridge 10% = 0.61734 (最佳) > VE alone 0.63287 > raw DeBERTa 0.63830
DeBERTa >> 树模型 DeBERTa LoRA 单折 (0.638) 远超所有树模型 ensemble (0.664)。Transformer 语义理解是核心竞争力。
OOF RMSE ≠ Kaggle RMSE OOF=1.117 → Kaggle=0.638，差距 0.48。测试集比训练集更容易预测。不能只优化 OOF。
弱模型会伤害集成 添加 OOF>1.2 的弱模型反而降低 Kaggle 分数。只集成 OOF<1.15 的模型。

原理分析:

  • Transformer 回归模型倾向于输出"安全"的中间值，避免极端预测
  • 这是 softmax 归一化和 MSE 损失的固有特性 — 模型最小化期望误差时倾向于预测均值
  • CORAL ordinal loss 缓解了部分问题，但压缩仍然存在
  • 方差扩展是后处理校准，不改变模型训练，只调整输出分布

最佳混合比例: VE 90% + Ridge 10% (Ridge 提供稳定的基线预测，VE 提供语义信号)

失败尝试: 量化匹配 (Kaggle 0.851) 和 Power transform (Kaggle 0.752) 都过于激进
